# KKCODER 三端交互原理

## 三端是谁

```
手机（你的小米 14）
    ↕
云服务器（你的 VPS，跑 frp）
    ↕
电脑（你的开发机，跑 KKCODER 桌面端 + Claude Code）
```

- **手机**：遥控器，发指令、看结果
- **电脑**：干活的地方，Claude Code 只能在这跑
- **云服务器**：传话筒，让手机和电脑在不同网络下也能连上

---

## 场景一：手机和电脑在同一 WiFi（局域网）

```
手机  ───────────────  电脑
       同一个路由器
```

手机直接用电脑的局域网 IP（比如 `192.168.1.100`）+ 端口连接。

云服务器不参与，数据不出家门，延迟最低。

---

## 场景二：手机用 4G，电脑在家（不同网络）

这是云服务器发挥作用的场景。

### 问题

```
手机（4G 网络）  ✗ 连不上  电脑（家里宽带）
```

手机在公网，电脑在家里的内网（路由器后面），手机找不到电脑的地址。

### 解决：frp 内网穿透

```
手机（4G）
    ↓ 连接
云服务器（公网 IP: 1.2.3.4，端口 9527）   ← 手机连这里
    ↓ frp 隧道转发
电脑（家里宽带，内网 IP 192.168.1.100）   ← Claude Code 在这
```

**云服务器做了什么：**

1. 电脑上的 frp 客户端主动连到云服务器，建了一条隧道
2. 云服务器在公网开了个端口（比如 9527），等着手机来连
3. 手机连到 `1.2.3.4:9527`，云服务器把数据通过隧道转给电脑
4. 电脑的回复也通过隧道转回手机

**本质上：云服务器是一个"传话筒"。**

- 电脑主动连云服务器（因为电脑能上网，但外面找不到电脑）
- 手机也连云服务器（因为云服务器有公网 IP，手机能找到）
- 云服务器把两边的数据互相转

### 数据流

```
手机输入 "帮我写个函数"
    ↓ HTTP/WebSocket
云服务器（1.2.3.4:9527）
    ↓ frp 隧道
电脑上的 KKCODER 桌面端
    ↓ 写入 PTY
Claude Code 执行
    ↓ 输出结果
电脑上的 KKCODER 读取输出
    ↓ frp 隧道
云服务器
    ↓ WebSocket
手机显示结果
```

## 场景三：域名 + HTTPS（更安全）

```
手机（4G）
    ↓ https://kkcoder.example.com
Cloudflare / Nginx（你的云服务器）
    ↓ 反向代理 + TLS 终结
frp 隧道
    ↓
电脑
```

比场景二多了：
- 域名（不用记 IP）
- HTTPS 加密（数据传输不被窥探）
- 可以套 Cloudflare（隐藏真实 IP，防攻击）

---

## 桌面端做了什么（中间人角色）

```
┌─────────────────────────────────────┐
│          电脑上的 KKCODER            │
│                                     │
│  ┌──────────┐    ┌───────────────┐  │
│  │ axum 服务器│    │ Claude Code   │  │
│  │ (远程访问) │←──→│ (PTY 终端)    │  │
│  └─────┬────┘    └───────────────┘  │
│        │                            │
│   监听端口 9527                      │
│   接受手机的 WebSocket 连接          │
│   转发输入/输出                      │
└─────────────────────────────────────┘
```

桌面端同时干两件事：
1. **本地**：管理 Claude Code 的 PTY 进程（创建、输入、读取输出）
2. **远程**：开一个 Web 服务器，让手机可以通过网络操控上面的 PTY

手机发的每一条命令，桌面端都会写入对应的 PTY；Claude Code 的每一条输出，桌面端都会通过 WebSocket 发回手机。

---

## 手机端做了什么

手机端就是一个"远程遥控器"：

1. **配对**：告诉桌面端"我是合法设备"（PIN → Token）
2. **选会话**：看到有哪些 Claude Code 终端在跑，选一个进去
3. **输入**：把用户打的字发给桌面端
4. **显示**：把桌面端传回来的输出显示在屏幕上

手机本身不执行任何 Claude Code 的逻辑，只负责"看"和"按"。

---

## 一句话总结

**手机是遥控器，云服务器是传话筒，电脑上的 KKCODER 是中间人，Claude Code 是干活的。**

同 WiFi 时手机直接喊中间人，不同网络时通过云服务器传话。
